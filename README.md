# Nexus Code v3.0 — C++ Edition

Быстрый автономный кодинг-агент на C++17. Один бинарник, мгновенный запуск,
90 встроенных инструментов и 20 слэш-команд. Полностью русскоязычный интерфейс.

## «Монстро»-возможности

- **repo_map** — компактная карта кодовой базы (файлы + сигнатуры функций/типов по
  языкам, с фокус-ранжированием). Агент понимает большой проект, не читая всё подряд.
- **spawn_agent** — параллельная делегация read-only разведки суб-агентам с
  изолированным контекстом (возвращают выжимку, не засоряя главный ход).
- **multi_edit** — пачка правок одного файла атомарно (всё или ничего) за один вызов.
- **edit_file** — при промахе точного совпадения включается фоллбэк, толерантный к
  отступам; `read_file` нумерует строки для точного прицела.
- **ripgrep-бэкенд** — `grep`/`search_code`/`find_file` уважают `.gitignore` и не
  тонут в `build/`, `node_modules`, `.git` (откат на grep/find, если нет `rg`).
- **Семантическая память** — если задан `gemini_api_key`, факты индексируются эмбеддингами
  (`gemini-embedding-001`, 768d) и ищутся **гибридно** (семантика + FTS): находит по смыслу,
  а не только по общим словам (синонимы, перефраз). Плюс дедуп похожих фактов и `memory_pin`
  (закрепить важное — всегда всплывает). Без ключа — прежний лексический поиск.
- **Генерация картинок** — `generate_image`: по текстовому промпту рисует изображение
  (бесплатно, без ключа — Pollinations/Flux) и сохраняет на диск; `open_path` откроет
  результат в просмотрщике. Пример: `нарисуй логотип: фиолетовый лис, плоский вектор`.
- **Зрение (картинки)** — впиши в запрос путь к картинке или URL (`.png/.jpg/.jpeg/.gif/.webp/.bmp`),
  и Nexus приложит её к сообщению как изображение (локальные → data:base64, ссылки → как есть).
  Работает на vision-моделях (Gemini, GPT-4o, *-vl и т.п.). На **текстовой модели (DeepSeek)**
  включается **vision-мост**: картинку описывает Gemini (по `gemini_api_key`), и её описание
  уходит в основную модель текстом — так распознавание работает на любом бэкенде.
  Пример: `что не так на этом скрине ~/Desktop/bug.png`.
- **MCP-клиент** — подключение внешних tool-серверов по Model Context Protocol
  (stdio/JSON-RPC 2.0). Их инструменты вливаются в реестр как `mcp__<server>__<tool>`.
  Настраивается в `~/.nexuscode/mcp.json` (стандартный блок `mcpServers`), статус — `/mcp`.

### MCP-серверы (`~/.nexuscode/mcp.json`)

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projects"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  }
}
```

Nexus поднимает серверы при старте, выполняет handshake `initialize`/`tools/list` и
регистрирует их инструменты. Схема параметров чистится под строгие эндпоинты (Gemini).
Инструменты с `annotations.readOnlyHint` считаются read-only (параллелятся, без переспроса),
остальные — мутирующими (подтверждение + последовательно). Команда `/mcp` показывает статус.

## Возможности

- **Function calling** в формате OpenAI (`api.cpp`).
- **Самодостаточная сборка** — SQLite (amalgamation) и nlohmann/json вшиты в
  `third_party/`. Никаких `-dev` пакетов ставить не нужно.
- **Авто-выбор HTTP-бэкенда** — `libcurl`, если есть его dev-заголовки, иначе
  откат на CLI `curl(1)`. Оба варианта работают сразу.
- **Векторная память** — локально, мешок слов + косинусное сходство, без
  внешних сервисов (понимает UTF-8/кириллицу).
- **История в SQLite** — сессии, снимки (checkpoints), задачи.
- **90 инструментов**: файлы, поиск (+repo_map), система, серверы, сеть, git, деплой,
  код, планирование (+spawn_agent), безопасность, утилиты.

## Требования

- CMake ≥ 3.16, компилятор C++17 (g++/clang++)
- `curl` в PATH (используется как HTTP-бэкенд, если нет dev-заголовков libcurl)

## Сборка и установка

```bash
./install.sh                 # собирает и ставит в ~/.local/bin
# или вручную:
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

## Настройка

Ключ берётся из (в порядке приоритета): переменной окружения `NEXUS_API_KEY`
или файла `~/.nexuscode/config.json`.

```bash
export NEXUS_API_KEY=sk-...
# или:
nexus
› /config set api_key sk-...
```

Ключи в `~/.nexuscode/config.json`: `api_key`, `base_url`, `model`, `mode`,
`temperature`, `max_iterations` (0 = без лимита), `auto_approve`, `log_level`.

## Запуск

```bash
nexus                       # интерактивная сессия (REPL)
nexus -p "<запрос>"         # один запрос и выход
nexus --auto "<запрос>"     # без подтверждений на изменяющие действия
nexus --config              # показать текущие настройки
```

### Слэш-команды

`/help /config /model /mode /auto /tools /stats /memory /forget /plan /execute
/search /run /deploy /checkpoint /checkpoints /undo /rollback /clear /exit`

## Архитектура

| Файл | Назначение |
|------|------------|
| `main.cpp` | точка входа, разбор аргументов, REPL |
| `agent.cpp` | цикл «модель + вызовы инструментов» |
| `api.cpp` | клиент chat completions |
| `http.cpp` | HTTP-бэкенд (libcurl или curl CLI) |
| `tools.cpp` | реестр 90 инструментов (реализации в `src/tools/*.cpp`) |
| `memory.cpp` | локальная векторная память |
| `history.cpp` | SQLite: сессии, снимки, задачи |
| `commands.cpp` | обработка слэш-команд |
| `config.cpp` | загрузка/сохранение JSON-конфига |
| `logger.cpp` | логирование в файл |
| `util.cpp` | общие хелперы |

Данные хранятся в `~/.nexuscode/` (`config.json`, `nexus_cpp.db`, `logs/agent.log`).
